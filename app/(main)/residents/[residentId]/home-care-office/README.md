# 居宅介護支援事業所タブ設計書

画面名: `居宅介護支援事業所タブ`  
パス: `/residents/[residentId]` (詳細情報タブ内)  
URL: https://carebase-staff.vercel.app/residents/[residentId]

## 概要

利用者詳細画面内の居宅介護支援事業所タブ設計書です。  
利用者の居宅介護支援事業所情報の表示、登録、編集、削除機能を提供します。  
1利用者につき複数事業所の登録を可能にし、事業所名、ケアマネージャー名、連絡先情報を効率的に管理できます。

### 画面構成

- タブヘッダー（居宅介護支援事業所タブ選択状態）

### アクション

| 項目名     | 処理内容                             | 対象API                                              | 遷移先画面                 |

### バリデーション仕様

#### 入力形式

- **電話番号・FAX番号**: 数字、ハイフン、プラス、括弧、スペースのみ許可
- **事業所名**: 1文字以上100文字以内
- **ケアマネージャー名**: 1文字以上50文字以内
- **住所**: 1文字以上200文字以内
- **備考**: 任意入力

#### 表示制御

- 必須項目には赤いアスタリスク（\*）を表示
- エラー時は該当フィールドを赤枠で強調
- 送信中はフォーム全体を無効化
- ネットワークエラー時はリトライボタンを表示
- 1利用者につき1事業所の制限により、登録済み時は追加ボタンを非表示

#### エラーハンドリング

- バリデーションエラーの個別表示
- ネットワークエラーの統一表示
- 削除処理エラーの表示
- 楽観的更新とエラー時のロールバック
- 事業所重複登録の防止

## 業務要件

### 機能要件

- 居宅介護支援事業所の情報を表示・管理
- 事業所名、ケアマネージャー名の管理
- 連絡先情報（電話番号、FAX、住所等）の管理
- 1利用者につき1事業所の登録

### 制約事項

- 1利用者につき1つの居宅介護支援事業所のみ登録可能
- 事業所情報は必須項目（事業所名、ケアマネージャー名、電話番号、住所）の入力が必要
- 削除時は確認モーダルによる二段階確認が必要
- ネットワークエラー時のリトライ機能を提供

### データ整合性

- 利用者詳細画面の他のタブとの情報連携
- 親Issue #133の利用者詳細画面仕様との整合性確保
- 既存のCRUD操作パターンとの統一

## 技術仕様

### データ型定義

```typescript
// HomeCareOfficeFormData型（types/resident-data.ts）
interface HomeCareOfficeFormData {
  businessName: string; // 事業所名（必須、100文字以内）
  careManager: string; // ケアマネージャー名（必須、50文字以内）
  phone: string; // 電話番号（必須）
  fax?: string; // FAX番号（任意）
  address: string; // 住所（必須、200文字以内）
  notes?: string; // 備考（任意）
}
```

### サービス層

```typescript
// residentDataService.ts
- createHomeCareOffice(residentId, data): 新規登録
- updateHomeCareOffice(residentId, officeId, data): 更新
- deleteHomeCareOffice(residentId, officeId): 削除
```

### コンポーネント構成

```
居宅介護支援事業所タブ
├── HomeCareOfficeCard（事業所情報表示）
│   ├── 事業所基本情報表示
│   ├── 編集ボタン → HomeCareOfficeModal（編集モード）
│   └── 削除ボタン → GenericDeleteModal
├── 追加ボタン → HomeCareOfficeModal（作成モード）
└── HomeCareOfficeModal
    └── HomeCareOfficeForm（入力フォーム）
```

### 状態管理

- モーダル表示状態の管理
- フォーム送信中状態の管理
- エラー状態の管理
- 楽観的更新による即座のUI反映
